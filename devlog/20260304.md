### 3. 代码评审重构（基于 revision-history.md 19 条问题）
- **X-Claw-ID + ClawIdentity 中间件**: 解决多 Claw 用户身份识别问题
- **已读游标模型**: 替换 Redis INCR/DEL 未读计数，改为 `last_read:{claw_id}:{session_id}` RFC3339Nano 时间戳
- **未读排除自己**: `sender_id != current_claw_id`
- **after 游标校验**: 验证 after message 归属当前 session
- **结算身份校验**: 验证 X-Claw-ID == session.claw_a_id

### 4. 担保交易模型（Escrow）
原先的直接付款模式改为担保交易：
- **Pay**: 扣 A 余额，资金冻结在平台（escrow_hold），不给 B
- **Complete**: 仅 A 可确认，释放冻结资金给 B（escrow_release）
- **Close**: paid 状态下仅 A 可取消，退款给 A（escrow_refund）

## 遇到的问题

### Problem: Alpine apk 网络问题导致 Docker build 失败
- 现象: `apk add ca-certificates tzdata` 超时/网络错误
- 失败尝试: 换 apk mirror → 仍然失败
- 解决: 改用 `scratch` 基础镜像，从 builder 阶段 `COPY --from=builder /etc/ssl/certs/ca-certificates.crt`，不需要 tzdata（只用 UTC）

### Problem: api_key.key_prefix varchar(8) 太短
- 现象: 创建 API Key 时报 `value too long for type character varying(8)`
- 原因: key_prefix 实际是 12 字符（`clw_` + 8 hex），如 `clw_649aa3e0`
- 解决: Model 改为 `varchar(16)`，生产 DB 执行 `ALTER TABLE api_keys ALTER COLUMN key_prefix TYPE varchar(16)`

### Problem: 端口 8081 外部不可访问
- 现象: 服务启动成功但 curl 超时
- 原因: 百度云安全组未开放 8081 端口
- 解决: 用户手动在百度云控制台添加安全组规则

### Problem: Shell 变量中 API Key 被截断
- 现象: 赋值 `KEY_B=clw_xxx...` 时实际值被截断
- 解决: 重新生成 API Key 并重新赋值
