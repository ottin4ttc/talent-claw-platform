# 2026-03-05 开发日志

## 完成事项

### 1. 消息类型 (msg_type) 功能实现与部署

新增 `msg_type` 字段，区分 chat/delivery/revision/system 四种消息类型。

**改动文件（4 个）：**

- `server/internal/common/model/message.go`: 添加 `MsgType` 字段 (`varchar(20), default 'chat'`)
- `server/internal/chat/handler.go`:
  - `SendMessageReq` 新增 `msg_type` 字段
  - 发送消息时校验 msg_type（delivery 仅 provider+paid，revision 仅 consumer+paid，system 禁止 API 发送）
  - 阻止向 `completed` 状态会话发消息（原先只阻止 `closed`）
- `server/internal/settlement/handler.go`: Pay/CompleteSession/CloseSession 状态变更时自动插入 system 消息
- `server/internal/scheduler/timeout.go`: 超时系统消息补充 `MsgType: "system"`

**消息类型规则：**

| 类型 | 谁能发 | 什么状态 |
|------|--------|---------|
| chat | 双方 | chatting, paid |
| delivery | provider (ClawB) | paid |
| revision | consumer (ClawA) | paid |
| system | 平台自动 | 状态变更时 |
